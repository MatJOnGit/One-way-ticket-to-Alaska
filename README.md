-FR-

Ce projet, dont l'objectif est de développer un moteur de blog afin de permettre à un écrivain, Jean Forteroche, de publier en ligne son roman par épisode. Il s'agit du quatrième projet à réaliser dans le cadre du parcours de développeur web junior d'OpenClassRooms.

Définition du projet :

L'application doit comprendre deux interfaces : une interface frontend, essentiellement pour la lecture des billets, et une interface backend pour l'administration des billets.
Les billets doivent permettre l'ajout de commentaires, qui pourront être modérés dans l'interface administrateur. Les lecteurs pourront signaler des commentaires afin qu'ils remontent plus facilement dans celle-ci.

Moyens techniques :

L'application doit être réalisé en PHP (sans utiliser de framework) avec une base de données MySQL. L'interface WYSIWYG permettant l'édition des billets doit présenter tous les éléments d'un CRUD à l'aide de TinyMCE.
De plus, le code doit être construit avec une architecture MVC et, autant que possible, en orienté objet.



-EN-

This project aims to develop a blog system allowing a writer, Jean Forteroche, to publish his novel through multiple episodes. This is also the fourth project of my training as Junior Web Developer at OpenClassRooms.

Project definiton :

The app must contain two interfaces : a frontend interface, esentially for posts reading, and a backend interface for posts.
Tickets must allow users to add comments, which can be moderated in the admin interface. Readers will be able to post comments so that they can be traced back more easily.

Technical means :

The app must be developed in PHP (without using any framework) with a MySQL database. The WYSIWYG interface for ticket editing must present all elements of a CRUD, using TinyMCE.
In addition, the code must be built with an MVC architecture and, as much as possible, object-oriented.
